## 仓库目录
- `README.md`：项目入口说明
- `design/`：设计文档目录
  - `model.md`：CuKee 模型架构设计
  - `mortal.md`：Mortal 项目设计参考梳理

## 开发规则
- 开发需求先切换到 `main` 分支，拉取最新 `main` 分支，然后创建并切换到 `codex/` 开头的分支，在 `codex/` 分支进行开发。
- 编辑已有文件时保持该文件原有换行符风格，不要无意义地混用或转换行尾。
- 当前主要 Markdown 文档使用 `CRLF` 行尾，尤其是 `README.md` 和 `design/model.md`。
- 新增 Markdown 文档优先使用 `CRLF` 行尾，除非所在目录已有更明确的约定。
